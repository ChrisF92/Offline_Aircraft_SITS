Offline Aircraft SITS — Build Instructions (v1.1.0)
==================================================

You maintain the .xlsm workbook locally from the .txt VBA sources in this folder.

Prerequisites
-------------
- Microsoft Excel 2016 (Office 16) with macros enabled
- Trust access to the VBA project object model (optional, for import tools):
    File > Options > Trust Center > Trust Center Settings > Macro Settings >
    check "Trust access to the VBA project object model"

Overview — master template workflow
-------------------------------------
1. Start from a polished **master template** (no aircraft tail).
2. **Polish layout on the Master in Excel** (column widths, button positions on
   Dashboard / SITS / Usage / Audit Log / Help). Day-to-day refresh does **not** reset
   these — only explicit Rebuild macros do.
3. For each aircraft: **File > Save As** a copy (e.g. `ZZ383_SITS_v2.xlsm`).
4. On the aircraft copy, run **Setup Wizard** to declare the tail and import data.
5. Use **Dashboard / SITS buttons** for day-to-day actions (log sorties, work packages,
   extensions, deferrals, etc.).
6. To wipe aircraft data later: Dev Tools → **Reset to Master** (layout preserved).

There is no Phase 0 / blank-workbook scaffold builder anymore. New Masters come from
an existing Master (or a Reset to Master copy). Layout tools rebuild presentation only.

Layout ownership
----------------
- **Rebuild** (explicit): default column widths, shell labels, and buttons.
- **Refresh** (day-to-day): data and status only — preserves widths and button
  placement; adds any *missing* action buttons at default anchors.
- **Rebuild Dashboard** / **Rebuild SITS Layout**: full/default layout reset when
  you intentionally want scaffolding again.
- **Ensure Missing Buttons**: adds new Dashboard buttons without moving existing ones;
  also removes the retired **Phase 0 Setup** button if still present.
- **Reset to Master**: clears aircraft data back to empty-master config while
  preserving all formatting.

-----------------------------
Open the VBA editor (Alt+F11). For each file below, insert a new standard module
(Insert > Module) if missing, set the module Name property exactly as shown, and
paste the file contents.

IMPORTANT: Module names must NOT be the same as a macro you call from another
module (e.g. do not name a module "Setup_CreateTables").

UserForms
---------
Log Sortie:
1. Insert > UserForm, Name = frmLogSortie
2. Add static shell controls per vba/forms/LOG_SORTIE_FORM_LAYOUT.txt
3. Insert > Class Module, Name = clsSortieUsageRow
   Paste vba/forms/clsSortieUsageRow.txt
4. Paste vba/forms/frmLogSortie.txt into the form code module

Work Packages:
1. Insert > UserForm, Name = frmWorkPackage
2. Add controls per vba/forms/WORK_PACKAGE_FORM_LAYOUT.txt
3. Paste vba/forms/frmWorkPackage.txt into the form code module

Extend / Defer / Horizon:
1. Create frmExtend, frmDefer, frmHorizon from the matching LAYOUT + code files
   (frmExtend needs lblCurrentExt — see EXTEND_FORM_LAYOUT.txt upgrade note)

Progress (busy dialog):
1. Insert > UserForm, Name = frmProgress
2. Add controls per vba/forms/PROGRESS_FORM_LAYOUT.txt (ShowModal = False)
3. Paste vba/forms/frmProgress.txt into the form code module
   Used by Refresh View / Log Sortie to show status without ghosting Excel.

Aircraft setup (Setup Wizard)
-----------------------------
1. Open the master template.
2. File > Save As > `<TAIL>_SITS_v2.xlsm` (e.g. ZZ383_SITS_v2.xlsm).
3. On the Dashboard, click **Setup Wizard**.
4. Enter tail number and import SITS Long Forecast, airframe usage, and engine usage.
5. Confirm — populates tblSITS, tblUsageBaseline, tblUsageCurrent.

Day-to-day actions
------------------
Log Sortie:
1. Click **Log Sortie** on the Dashboard (or macro RunLogSortie).
2. Enter sortie date and required HUMS Op Number.
3. In each group (Airframe / Engines / WARE), enter Usage for counters that
   changed on the sortie. Current Total and New Total are read-only.
   - Hours counters (DA, MX, XJ, HH, E1, E2, OH): enter h:mm (e.g. 1:30)
   - OEI second totals (1E/1G/2E/2G): enter decimal seconds (e.g. 3.38)
   - Number counters: enter a positive number
   - Leave Usage blank for counters with no change
4. Click Log sortie.

SITS view:
1. Open the **SITS** sheet after setup or import.
2. Click **Horizon** and set forecast hours / days / number to filter the view.
3. Use **Refresh View** / **Create Work Package** / **Manage Work Packages** /
   **Horizon** / **Extend** / **Defer** / **Add Task** / **Remove Task** /
   **Manual Update** buttons.
4. Use the **AutoFilter** dropdowns on the header row (row 7).
5. Life remaining highlighting:
   - Amber when due soon (within warn thresholds: default 10:00 hours / 7 days)
   - Pink/red when overdue
   - Light blue for zero-interval tasks needing manual update, or selected rows

Work packages:
1. Tick Selector checkboxes on SITS → Create Work Package (ORN ####TAILDDMMYY).
2. Manage Work Packages: Delete (undo), Remove task, or Close (reforecast).

Extend / Defer / Manual Update:
- Extend: scheduled tasks sharing Interval Type + Actual Interval + Life Remaining;
  form shows Current/Total Extension as amount (percent); SNOW + authoriser.
- Defer: exactly one PT2/PT3; HH due (h:mm) or date due (dd/mm/yyyy).
- Manual Update: zero Actual Interval tasks only; step-by-step prompts for due
  date or hours/number until due; clears the Needs Manual Update flag.

Dashboard
---------
  Open work packages / manual-update / warning / overdue counts
  Pre-setup / master: **Setup Wizard** + **View Help** under User Actions
  After setup: Log Sortie, Work Packages, View Audit Log, View Help
  Extend / Defer / Manual Update are on the **SITS sheet only**
  Import SITS and Refresh Usage live under Developer Tools
  Help sheet explains each part of the system (Rebuild Help Sheet)

Developer Tools (hidden by default)
-----------------------------------
  Alt+F8 → **EnableDevTools** to show; **HideDevTools** to hide.
  Import SITS, Refresh Usage, Repair Formats, Repair Visibility, Validate Structure,
  Rebuild Dashboard, Ensure Missing Buttons, Rebuild SITS Layout,
  Format Usage Matrix, Rebuild Help Sheet, Protect Workbook, Unprotect Workbook,
  Hide Dev Tools, Reset to Master

Reset to Master (data only — preserves formatting)
--------------------------------------------------
  Dashboard Dev Tools → **ResetWorkbookToMaster** (module modReset)
  Clears SITS / usage / work packages / sortie log / audit / usage snapshot data
  and reseeds master config (no tail, SetupComplete=False, MasterTemplate=True).
  Does **not** rebuild sheets, column widths, colours, or button placement.
  Prefer **Ensure Missing Buttons** after pasting VBA so new buttons appear
  without wiping your Master layout.

Workbook protection
-------------------
Sheets and workbook structure are protected so users cannot edit cells or
unhide/rename/delete sheets, while day-to-day macros and forms keep working.

  - Every sheet is protected with `UserInterfaceOnly:=True`
  - Form Control buttons and Selector checkboxes stay clickable
  - Workbook structure protection stops unhiding data sheets
  - `Workbook_Open` re-applies protection
  - Mutating macros call `SuspendWorkbookProtection` / `ResumeWorkbookProtection`
  - Admin: **ProtectWorkbook** / **UnprotectWorkbook** (password `PROTECT_PASSWORD`
    in CONFIG, default `SITSv2` — not a security boundary)

Still editable by users:
  - SITS Selector checkboxes
  - AutoFilter on the SITS header row
  - Horizon / other settings via UserForms

SITS life remaining model
-------------------------
GOLDesp life remaining at import is the **baseline**. After import, each sortie
increments usage; life remaining uses usage delta since the import snapshot:

  Life Remaining = Baseline Life Remaining − (Current Usage − Snapshot Usage)

Current usage is looked up from tblUsageCurrent by **interval type code only**.
DD calendar tasks prefer Projected Due Date when present.

Import rules (strict — abort if any required column missing)
------------------------------------------------------------
SITS Long Forecast:
  Interval Type Code, Life Remaining, Task Code, Task Sequence,
  Task Description, Serial, Actual Interval, Extension, End Item Serial
  Task codes beginning with Gen/ are skipped.
  Number-based interval types (excluding DD) are skipped on import unless both
  Actual Interval and Life Remaining are zero.

Usage exports (airframe + engine):
  Interval Type, Task Code, Life Used, Top Serial

Engine usage must include SM and SN interval codes after merge
(typically WC/STARTS/ECU on P4N155/P4N175).

First release — v1.0.0 (migrating from pre-release builds)
---------------------------------------------------------
Pre-release builds used internal numbering (up to 2.3.8) and included Phase 0
scaffold macros. **1.0.0** is the first release: Phase 0 sources are gone;
data reset remains via **Reset to Master**.

1. **First** delete retired modules from the VBA project if present:
   modSetupSheets, modSetupTables, modSetupRunAll
   (Leaving them in place after updating modUtility causes compile errors such as
   "Sub or Function not defined: ReuseSheetAs" until they are removed.)
2. Replace / paste updated modules:
   CONFIG, modUtility, modSITS, ThisWorkbook,
   Setup_CreateConfig, Setup_CreateNamedRanges, Setup_CreateDashboard,
   Setup_FormatUsageMatrix, Setup_CreateHelpSheet
3. Insert module **modReset** from `vba/modules/modReset.txt`
   (Name property = modReset). This owns `ResetWorkbookToMaster`.
4. Compile VBAProject.
5. Alt+F8 → **EnableDevTools** → **Ensure Missing Buttons**
   (removes Phase 0 Setup button; adds/compacts remaining Dev Tools).
   Or **Rebuild Dashboard** if you want a full default layout.
6. Optional: **Rebuild Help Sheet** so operator guide text matches v1.0.0.
7. Smoke-test: Reset to Master → Setup Wizard on a Save As copy.

Note: modUtility still includes legacy sheet helpers (`ReuseSheetAs`, etc.) so a
partial paste does not brick compile mid-upgrade. New code does not call them;
delete the retired Phase 0 modules and ignore those helpers.

Upgrade — v1.1.0 (from v1.0.0)
------------------------------
1. Paste updated modules: CONFIG, Setup_CreateDashboard, Setup_CreateHelpSheet,
   modImportSITS, modExtensions, modSITS, modSITSEdit.
2. Update frmExtend: add Label `lblCurrentExt`, set ColumnCount=4 on lstTasks,
   paste `vba/forms/frmExtend.txt` (see EXTEND_FORM_LAYOUT.txt).
3. Compile VBAProject.
4. Alt+F8 → **EnableDevTools** → **Ensure Missing Buttons** (adds Manual Update
   on SITS; fixes View Help placement after Setup Wizard).
5. Optional: **Rebuild Help Sheet**.

Pre-release notes (former 2.3.x)
--------------------------------
Historical Phase 0 / Setup_RunAll / Setup_CreateSheets / Setup_CreateTables steps
are obsolete. For older workbooks still on those builds:

1. Bring VBA up to late pre-release behaviour first if needed (protection, Help sheet,
   Extend/Defer, Usage matrix, Dev Tools), using a known-good Master as the base.
2. Then apply the v1.0.0 steps above, then v1.1.0.
3. Named ranges: **Rebuild Dashboard** now refreshes them automatically
   (or run Setup_CreateNamedRanges).
4. Help sheet: Dev Tools → Rebuild Help Sheet.
5. Usage matrix: Format Usage Matrix once if the sheet still uses an old list layout;
   afterwards Refresh Usage / Log Sortie keep it current.

Troubleshooting
---------------
"Expected variable or procedure, not module"
  -> Module Name must be modSetupDashboard etc., not the macro name.

"Variable not defined: frmLogSortie" / other forms
  -> Create the UserForm in the IDE. The form object must exist even though
     the code is stored as a .txt file in the repo.

UserForm opens on the wrong monitor
  -> Forms call CenterUserFormOnExcel (modUtility) from UserForm_Activate.

"Missing named range: Open_WP_Count"
  -> Rebuild Dashboard (or Setup_CreateNamedRanges).

VBA name shadowing ("expects an array" on a function call)
  -> Do not name parameters the same as functions (VBA is case-insensitive).

All worksheets stuck hidden
  -> Repair Visibility, then FinalizeSetupSheetLayout / ProtectWorkbook.

Cannot edit cells / cannot unhide sheets
  -> Expected when workbook protection is on. Use UnprotectWorkbook for Master
     layout work, then ProtectWorkbook again.

Runtime error 1004 during Setup Wizard / Import SITS ("Table features aren't
available because the sheet is protected")
  -> Import and other table-mutating macros temporarily suspend protection.
     Re-import modUtility + import / wizard modules if an older paste still fails.

SITS Selector checkboxes do nothing while sheet is protected
  -> Refresh SITS re-applies LinkedCell unlocks. Re-import modUtility + modSITS.

Dashboard C6/C7 stay "stale" (strikethrough)
  -> Rebuild Dashboard or Alt+F8 → RepairDashboardConfigFormulas (or press F9).

Serial numbers show as 1.11E+12 or lose leading zeros
  -> Repair Formats, or re-import SITS/usage after updating modUtility.

Time values calculate incorrectly / show as 86.61 or plain text
  -> Hour counters are day fractions with [h]:mm. Use Repair Formats.

SITS life remaining does not change after sorties
  -> Re-import SITS or run Setup Wizard to refresh tblUsageSnapshot.
  -> Run RunRefreshSITS on the SITS sheet.

Dashboard buttons missing after updating VBA
  -> Ensure Missing Buttons (adds only what is missing).
  -> Rebuild Dashboard only if you want to reset column widths / placement.

Column widths / buttons reset after Refresh
  -> Refresh is data-only. Avoid Rebuild Dashboard unless intended.

Audit Log columns resize after each log entry
  -> Re-import updated modAuditTrail (preserves widths around each append).

ORN rejected on create
  -> Must be ####TAILDDMMYY with this aircraft's tail and a real DDMMYY date.

Extend rejected — mixed intervals
  -> All ticked tasks must share Interval Type Code, Actual Interval, and Life Remaining.

Extend rejected — PT2/PT3
  -> Use Defer for deferred faults.

Manual Update rejected — non-zero interval
  -> Only Actual Interval = 0 tasks can be updated via Manual Update.

Defer rejected — more than one task
  -> Defer exactly one PT2/PT3 at a time.

Compile error: Sub or Function not defined (Setup_RunAll / ReuseSheetAs / AddSetupSheet)
  -> You still have retired Phase 0 modules in the VBA project.
     Delete modSetupSheets, modSetupTables, and modSetupRunAll, then compile again.
     Use Reset to Master for data wipe. Ensure Missing Buttons removes the Phase 0
     Dashboard button.
