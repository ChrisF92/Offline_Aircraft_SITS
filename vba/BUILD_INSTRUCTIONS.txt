Offline Aircraft SITS v2 — Build Instructions (Phase 3)
=========================================================

You build the .xlsm workbook locally from the .txt VBA sources in this folder.

Prerequisites
-------------
- Microsoft Excel 2016 (Office 16) with macros enabled
- Trust access to the VBA project object model (optional, for import tools):
  File > Options > Trust Center > Trust Center Settings > Macro Settings >
  check "Trust access to the VBA project object model"

Overview — master template workflow
-------------------------------------
1. Build the **master template once** (Phase 0 setup — no aircraft tail).
2. Save as `Wildcat_SITS_Master.xlsm` (or similar).
3. For each aircraft: **File > Save As** a copy (e.g. `ZZ383_SITS_v2.xlsm`).
4. On the aircraft copy, run **Setup Wizard** to declare the tail and import data.
5. Use **Dashboard buttons** for day-to-day actions (log sorties, re-import SITS, etc.).

Step 1 — Create a new workbook
------------------------------
1. Open Excel and create a blank workbook.
2. Save As > Excel Macro-Enabled Workbook (.xlsm)
   Example for master: Wildcat_SITS_Master.xlsm

Step 2 — Import VBA modules (order recommended)
-----------------------------------------------
Open the VBA editor (Alt+F11). For each file below, insert a new standard module
(Insert > Module), set the module Name property exactly as shown, and paste the
file contents.

IMPORTANT: The module Name must match the Attribute VB_Name line at the top of
each file. Module names must NOT be the same as a macro you call from another
module (e.g. do not name a module "Setup_CreateTables" — that causes a compile
error "Expected variable or procedure, not module").

Runtime modules:

  File                                  Module Name (Name property)
  ------------------------------------  ---------------------------
  vba/modules/CONFIG.txt                CONFIG
  vba/modules/modUtility.txt            modUtility
  vba/modules/modValidation.txt         modValidation
  vba/modules/modImportValidation.txt   modImportValidation
  vba/modules/modAuditTrail.txt         modAuditTrail
  vba/modules/modImportSITS.txt         modImportSITS
  vba/modules/modImportUsage.txt        modImportUsage
  vba/modules/modSetupWizard.txt        modSetupWizard
  vba/modules/modUsageLog.txt           modUsageLog
  vba/modules/modSITS.txt               modSITS

Setup modules (one-time use):

  File                                  Module Name
  ------------------------------------  -------------
  vba/setup/Setup_CreateSheets.txt      modSetupSheets
  vba/setup/Setup_CreateTables.txt      modSetupTables
  vba/setup/Setup_CreateConfig.txt      modSetupConfig
  vba/setup/Setup_CreateNamedRanges.txt modSetupNames
  vba/setup/Setup_CreateDashboard.txt   modSetupDashboard
  vba/setup/Setup_RunAll.txt            modSetupRunAll

The macros you run still use their procedure names, e.g. Setup_CreateTables and
Setup_RunAll — only the module container names differ.

ThisWorkbook code:

  vba/modules/ThisWorkbook.txt        -> Paste into ThisWorkbook object (not a module)

Step 3 — Create the Log Sortie UserForm (Phase 2)
-------------------------------------------------
1. In the VBA editor: Insert > UserForm
2. Set the form Name property to: frmLogSortie
3. Add controls per vba/forms/LOG_SORTIE_FORM_LAYOUT.txt (names must match)
4. Paste the code from vba/forms/frmLogSortie.txt into the form's code module
   (omit any VERSION/Begin block if the IDE already generated form metadata)

Step 4 — Run Phase 0 setup (master template — once)
---------------------------------------------------
Required before running setup (compile-time dependencies):
  CONFIG, modUtility, modValidation

Import modAuditTrail before normal use (Workbook_Open / setup wizard logging).

1. Run **Setup_RunAll** (Dashboard dev button or macro in modSetupRunAll).
2. Do **not** enter a tail number — the master template is aircraft-neutral.
3. Run **Validate Structure** on the Dashboard to confirm structure.
4. Save as your master copy (e.g. Wildcat_SITS_Master.xlsm).

Step 5 — Create an aircraft workbook
------------------------------------
1. Open the master template.
2. File > Save As > `<TAIL>_SITS_v2.xlsm` (e.g. ZZ383_SITS_v2.xlsm).
3. On the Dashboard, click **Setup Wizard**.
4. Enter tail number and import SITS Long Forecast, airframe usage, and engine usage.
5. Confirm — populates tblSITS, tblUsageBaseline, tblUsageCurrent.

Step 6 — Log sorties (Phase 2)
------------------------------
1. Click **Log Sortie** on the Dashboard (or macro RunLogSortie).
2. Enter sortie date and optional sortie reference.
3. Select a usage counter from the baseline list, enter increment, click Add.
- Hours counters: enter h:mm (e.g. 1:30) — stored as [h]:mm
- OEI second totals (1E/1G/2E/2G): enter decimal seconds (e.g. 3.38) — stored as Number
- Other m:ss elapsed counters: enter m:ss — stored as [m]:ss
- Number counters: enter a positive number
4. Add all increments for the sortie, then click Log sortie.

Each logged increment:
  - Appends a row to tblUsageLog
  - Updates tblUsageCurrent
  - Refreshes the Usage sheet view
  - Recalculates and refreshes the SITS view (Phase 3)

Step 7 — SITS view and refresh (Phase 3)
------------------------------------------
1. Open the **SITS** sheet after setup or import.
2. Set **Forecast hours** (B3) and **Forecast days** (B4) to filter the view:
   - HH/E1/E2 tasks: shown only if life remaining is within the hour threshold
   - DD tasks: shown only if due within the day threshold
   - Other interval types: always shown
3. Assign macro **RunRefreshSITS** to a button on the SITS sheet (Insert > Button).
4. Life remaining warnings highlight rows in pink when below config thresholds
   (default 10:00 hours / 7 days).

SITS recalculation runs automatically after:
  - SITS import (Import SITS / Setup Wizard)
  - Sortie logging
  - Manual RunRefreshSITS

Dashboard buttons (created by Setup_RunAll / Rebuild Dashboard)
---------------------------------------------------------------
User Actions:
  Setup Wizard, Log Sortie, Import SITS, Refresh Usage, View Audit Log

Developer Tools:
  Repair Formats, Repair Visibility, Validate Structure, Rebuild Dashboard, Phase 0 Setup

Step 8 — Save
-------------
Save the workbook.

What Phase 3 includes
---------------------
- modSITS — baseline + usage-delta life remaining recalculation
- tblUsageSnapshot — usage per interval code at last SITS import/setup
- Baseline columns on tblSITS (hidden from view, used for delta maths)
- SITS sheet view linked to tblSITS with forecast filter
- Warning highlighting (default 10:00 hours / 7 days)
- PT2/PT3 import: extension reset to 0 / 0:00; life remaining from GOLDesp
- DD tasks: days-when-due with calendar elapsed time since import
- Auto-refresh after SITS import, setup, and sortie logging

What is NOT included yet
------------------------
- Work packages, extensions, deferrals (Phases 4–5)
- Selector editing (display-only until work packages)
- Sortie correction / reversal UI (CorrectedFrom column reserved)
- Forecast settings UserForm (values editable on SITS sheet B3/B4 for now)

SITS life remaining model (Phase 3)
-----------------------------------
GOLDesp life remaining at import is the **baseline** (trusted — may include
life accrued on other airframes). After import, each sortie increments usage;
life remaining is updated by **usage delta since the import snapshot**:

  Life Remaining = Baseline Life Remaining − (Current Usage − Snapshot Usage)

Current usage is looked up from tblUsageCurrent by **interval type code only**.
DD calendar tasks count down elapsed days since the SITS snapshot date when
GOLDesp provides a days count; absolute due dates are shown as imported.

Import rules (strict — abort if any required column missing)
------------------------------------------------------------
SITS Long Forecast:
  Interval Type Code, Life Remaining, Task Code, Task Sequence,
  Task Description, Serial, Actual Interval, Extension, End Item Serial

Usage exports (airframe + engine):
  Interval Type, Task Code, Life Used, Top Serial

Engine usage must include SM and SN interval codes after merge
(typically WC/STARTS/ECU on P4N155/P4N175).

  ZZ383 Item Usage.xlsx               — Airframe usage sample
  ZZ383 Engine Usage.xlsx             — Engine usage sample (includes SM/SN)

Upgrading an existing Phase 0–2 workbook to Phase 3
-----------------------------------------------------
1. Import the new modSITS module and update all changed .txt modules.
2. Re-run **Setup_CreateTables** (or Phase 0 Setup) to add baseline columns and
   tblUsageSnapshot — this clears data tables; prefer running on a fresh copy.
3. Re-run Setup Wizard or re-import SITS to establish baselines and snapshots.

Troubleshooting
---------------
"Expected variable or procedure, not module"
  -> Setup module Name must be modSetupRunAll etc., not Setup_RunAll.

"Variable not defined: frmLogSortie"
  -> Create the UserForm in the IDE (Step 3). The form object must exist even
     though the code is stored as a .txt file in the repo.

VBA name shadowing ("expects an array" on a function call)
  -> Do not name parameters the same as functions (VBA is case-insensitive).
     Example: use skipOverwritePrompt, not confirmOverwrite with ConfirmOverwrite.

All worksheets stuck hidden
  -> Click Repair Visibility on the Dashboard, or run RepairSetupSheetVisibility.

Serial numbers show as 1.11E+12 or lose leading zeros
  -> Re-import SITS/usage after updating modUtility, or click Repair Formats.
  -> Serial columns are stored as text (@) using the source cell display text.

Time values calculate incorrectly / show as 86.61 or plain text
  -> 86.61 is the internal day-fraction for 2078:43. Hour counters are stored
     as day fractions with [h]:mm (same principle as SITS), not as text.
  -> Click Repair Formats on the Dashboard, or re-run setup wizard import.
  -> HH/E1/E2 Format column will show Hours; Baseline and Usage show [h]:mm.

SITS life remaining does not change after sorties
  -> Re-import SITS or run Setup Wizard to refresh tblUsageSnapshot.
  -> Run RunRefreshSITS on the SITS sheet.

Dashboard buttons missing after updating VBA
  -> Click Rebuild Dashboard on the Dashboard (Developer Tools section).
