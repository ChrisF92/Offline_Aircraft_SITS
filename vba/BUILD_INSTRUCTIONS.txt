Offline Aircraft SITS v2 — Build Instructions (Phase 2)
=========================================================

You build the .xlsm workbook locally from the .txt VBA sources in this folder.

Prerequisites
-------------
- Microsoft Excel 2016 (Office 16) with macros enabled
- Trust access to the VBA project object model (optional, for import tools):
  File > Options > Trust Center > Trust Center Settings > Macro Settings >
  check "Trust access to the VBA project object model"

Step 1 — Create a new workbook
------------------------------
1. Open Excel and create a blank workbook.
2. Save As > Excel Macro-Enabled Workbook (.xlsm)
   Example: ZZ383_SITS_v2.xlsm

Step 2 — Import VBA modules (order recommended)
-----------------------------------------------
Open the VBA editor (Alt+F11). For each file below, insert a new standard module
(Insert > Module), set the module Name property exactly as shown, and paste the
file contents.

IMPORTANT: The module Name must match the Attribute VB_Name line at the top of
each file. Module names must NOT be the same as a macro you call from another
module (e.g. do not name a module "Setup_CreateTables" — that causes a compile
error "Expected variable or procedure, not module").

Runtime modules:

  File                                  Module Name (Name property)
  ------------------------------------  ---------------------------
  vba/modules/CONFIG.txt                CONFIG
  vba/modules/modUtility.txt            modUtility
  vba/modules/modValidation.txt         modValidation
  vba/modules/modImportValidation.txt   modImportValidation
  vba/modules/modAuditTrail.txt         modAuditTrail
  vba/modules/modImportSITS.txt         modImportSITS
  vba/modules/modImportUsage.txt        modImportUsage
  vba/modules/modSetupWizard.txt        modSetupWizard
  vba/modules/modUsageLog.txt           modUsageLog

Setup modules (one-time use):

  File                                  Module Name
  ------------------------------------  -------------
  vba/setup/Setup_CreateSheets.txt      modSetupSheets
  vba/setup/Setup_CreateTables.txt      modSetupTables
  vba/setup/Setup_CreateConfig.txt      modSetupConfig
  vba/setup/Setup_CreateNamedRanges.txt modSetupNames
  vba/setup/Setup_CreateDashboard.txt   modSetupDashboard
  vba/setup/Setup_RunAll.txt            modSetupRunAll

The macros you run still use their procedure names, e.g. Setup_CreateTables and
Setup_RunAll — only the module container names differ.

ThisWorkbook code:

  vba/modules/ThisWorkbook.txt        -> Paste into ThisWorkbook object (not a module)

Step 3 — Create the Log Sortie UserForm (Phase 2)
-------------------------------------------------
1. In the VBA editor: Insert > UserForm
2. Set the form Name property to: frmLogSortie
3. Add controls per vba/forms/LOG_SORTIE_FORM_LAYOUT.txt (names must match)
4. Paste the code from vba/forms/frmLogSortie.txt into the form's code module
   (omit any VERSION/Begin block if the IDE already generated form metadata)

Step 4 — Run Phase 0 setup
--------------------------
Required before running setup (compile-time dependencies):
  CONFIG, modUtility, modValidation

Import modAuditTrail before normal use (Workbook_Open / setup wizard logging).

1. Run Setup_RunAll (modSetupRunAll) or the individual Setup_* macros.
2. Enter the aircraft tail number when prompted (e.g. ZZ383).
3. Run ValidateWorkbookStructure to confirm structure.

Step 5 — Run Phase 1 setup wizard
---------------------------------
1. Run macro RunSetupWizard (modSetupWizard).
2. Import SITS Long Forecast, airframe usage, and engine usage exports.
3. Confirm — populates tblSITS, tblUsageBaseline, tblUsageCurrent.

Step 6 — Log sorties (Phase 2)
------------------------------
1. Run macro RunLogSortie (or ShowLogSortieForm) from the Dashboard.
2. Enter sortie date and optional sortie reference.
3. Select a usage counter from the baseline list, enter increment, click Add.
- Hours counters: enter h:mm (e.g. 1:30) — stored as [h]:mm
- Minutes counters (e.g. 1E OEI seconds): enter m:ss — stored as [m]:ss
- Number counters: enter a positive number
4. Add all increments for the sortie, then click Log sortie.

Each logged increment:
  - Appends a row to tblUsageLog
  - Updates tblUsageCurrent
  - Refreshes the Usage sheet view

Assign macros to Dashboard shapes/buttons:
  RunSetupWizard, RunLogSortie, ImportSITSFromFile, ViewAuditLog

Step 7 — Save
-------------
Save the workbook.

What Phase 2 includes
---------------------
- Log Sortie UserForm (frmLogSortie) — baseline-driven increment entry
- tblUsageLog append on each sortie
- tblUsageCurrent rolling totals
- Usage sheet refresh (RefreshUsageView)

What is NOT included yet
------------------------
- SITS calculations and view (Phase 3)
- Work packages, extensions, deferrals (Phases 4–5)
- Sortie correction / reversal UI (CorrectedFrom column reserved)

Import rules (strict — abort if any required column missing)
------------------------------------------------------------
SITS Long Forecast:
  Interval Type Code, Life Remaining, Task Code, Task Sequence,
  Task Description, Serial, Actual Interval, Extension, End Item Serial

Usage exports (airframe + engine):
  Interval Type, Task Code, Life Used, Top Serial

Engine usage must include SM and SN interval codes after merge
(typically WC/STARTS/ECU on P4N155/P4N175).

Sample files in repo root:
  20260706-Long Forecast_ZZ382.xlsx   — SITS export sample (tail ZZ382)
  ZZ383 Item Usage.xlsx               — Airframe usage sample
  ZZ383 Engine Usage.xlsx             — Engine usage sample (includes SM/SN)

Troubleshooting
---------------
"Expected variable or procedure, not module"
  -> Setup module Name must be modSetupRunAll etc., not Setup_RunAll.

"Variable not defined: frmLogSortie"
  -> Create the UserForm in the IDE (Step 3). The form object must exist even
     though the code is stored as a .txt file in the repo.

VBA name shadowing ("expects an array" on a function call)
  -> Do not name parameters the same as functions (VBA is case-insensitive).
     Example: use skipOverwritePrompt, not confirmOverwrite with ConfirmOverwrite.

All worksheets stuck hidden
  -> Run RepairSetupSheetVisibility in modUtility, then FinalizeSetupSheetLayout.

Serial numbers show as 1.11E+12 or lose leading zeros
  -> Re-import SITS/usage after updating modUtility, or run RepairWorkbookFormats.
  -> Serial columns are stored as text (@) using the source cell display text.

Time values calculate incorrectly
  -> Hours use [h]:mm (Excel day-fraction storage). Minutes use [m]:ss.
  -> HH/E1/E2 showing as 86.61 means Format was Number not Hours — run
     RepairWorkbookFormats (fixes Format column and applies [h]:mm display).
  -> Underlying 86.61 for 2078:43 is correct; the cell must use [h]:mm to display it.
