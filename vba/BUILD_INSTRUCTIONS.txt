Offline Aircraft SITS v2 — Build Instructions (Phase 5)
=========================================================

You build the .xlsm workbook locally from the .txt VBA sources in this folder.

Prerequisites
-------------
- Microsoft Excel 2016 (Office 16) with macros enabled
- Trust access to the VBA project object model (optional, for import tools):
    File > Options > Trust Center > Trust Center Settings > Macro Settings >
    check "Trust access to the VBA project object model"

Overview — master template workflow
-------------------------------------
1. Build the **master template once** (Phase 0 setup — no aircraft tail).
2. Save as `Wildcat_SITS_Master.xlsm` (or similar).
3. **Polish layout on the Master in Excel** (column widths, button positions on
   Dashboard / SITS / Usage / Audit Log). Day-to-day refresh does **not** reset
   these — only explicit Rebuild macros do.
4. For each aircraft: **File > Save As** a copy (e.g. `ZZ383_SITS_v2.xlsm`).
5. On the aircraft copy, run **Setup Wizard** to declare the tail and import data.
6. Use **Dashboard / SITS buttons** for day-to-day actions (log sorties, work packages,
   extensions, deferrals, etc.).

Layout ownership
----------------
- **Setup / Rebuild** (once): default column widths, shell labels, and buttons.
- **Refresh** (day-to-day): data and status only — preserves widths and button
  placement; adds any *missing* action buttons at default anchors.
- **Rebuild Dashboard** / **Rebuild SITS Layout**: explicit full/default layout
  reset when you intentionally want scaffolding again.
- **Ensure Missing Buttons**: adds new Dashboard buttons without moving existing ones.

Step 1 — Create a new workbook
------------------------------
1. Open Excel and create a blank workbook.
2. Save As > Excel Macro-Enabled Workbook (.xlsm)
   Example for master: Wildcat_SITS_Master.xlsm

Step 2 — Import VBA modules (order recommended)
-----------------------------------------------
Open the VBA editor (Alt+F11). For each file below, insert a new standard module
(Insert > Module), set the module Name property exactly as shown, and paste the
file contents.

IMPORTANT: The module Name must match the Attribute VB_Name line at the top of
each file. Module names must NOT be the same as a macro you call from another
module (e.g. do not name a module "Setup_CreateTables" — that causes a compile
error "Expected variable or procedure, not module").

Step 4 — Run Phase 0 setup (master template — once)
---------------------------------------------------
Required before running setup (compile-time dependencies):
  CONFIG, modUtility, modValidation

Import modAuditTrail / modWorkPackage / modAssociatedTasks / modExtensions
before normal use.

1. Run **Setup_RunAll** (Dashboard dev button or macro in modSetupRunAll).
2. Do **not** enter a tail number — the master template is aircraft-neutral.
3. Run **Validate Structure** on the Dashboard to confirm structure.
4. Save as your master copy (e.g. Wildcat_SITS_Master.xlsm).

Step 5 — Create an aircraft workbook
------------------------------------
1. Open the master template.
2. File > Save As > `<TAIL>_SITS_v2.xlsm` (e.g. ZZ383_SITS_v2.xlsm).
3. On the Dashboard, click **Setup Wizard**.
4. Enter tail number and import SITS Long Forecast, airframe usage, and engine usage.
5. Confirm — populates tblSITS, tblUsageBaseline, tblUsageCurrent.

Step 6 — Log sorties
---------------------
1. Click **Log Sortie** on the Dashboard (or macro RunLogSortie).
2. Enter sortie date and required HUMS Op Number.
3. In each group (Airframe / Engines / WARE), enter Usage for counters that
   changed on the sortie. Current Total and New Total are read-only.
   - Hours counters (DA, MX, XJ, HH, E1, E2, OH): enter h:mm (e.g. 1:30)
   - OEI second totals (1E/1G/2E/2G): enter decimal seconds (e.g. 3.38)
   - Number counters: enter a positive number
   - Leave Usage blank for counters with no change
4. Click Log sortie.

Each entered usage value:
  - Appends a row to tblUsageLog
  - Updates tblUsageCurrent
  - Refreshes the Usage sheet view
  - Recalculates and refreshes the SITS view

Catalog codes not yet present in the usage baseline appear as (n/a) with
Usage locked until imported (e.g. WARE IL/IR/HE/FT on older exports).

Step 7 — SITS view and refresh (Phase 3)
------------------------------------------
1. Open the **SITS** sheet after setup or import.
2. Set **Forecast hours** (B3), **Forecast days** (B4), and **Forecast number** (B5) to filter the view:
   - HH/E1/E2 tasks: shown only if life remaining is within the hour threshold
   - DD tasks: shown only if due within the day threshold
   - Other number-based interval types (starts, cycles, etc.): shown only if life remaining is within the number threshold
   - Set any forecast to 0 to show all tasks of that type
3. Use **Refresh View** / **Create Work Package** / **Manage Work Packages** /
   **Extend** / **Defer** buttons on the SITS sheet (recreated by RefreshSITS).
4. Use the **AutoFilter** dropdowns on the header row (row 7) to show/hide interval
   types (e.g. hide AU, OP) or other columns — standard Excel filtering.
5. Life remaining highlighting:
   - Amber when due soon (within warn thresholds: default 10:00 hours / 7 days)
   - Pink/red when overdue (life remaining / days remaining below zero)
6. Zero-interval tasks awaiting manual update after WP close highlight in light blue.
7. Ticking a Selector checkbox highlights that row light blue; unticking restores the
   previous warning colour (or clears the fill). Refresh View once after importing
   updated VBA so checkboxes pick up the OnAction (SheetChange is a backup path).

SITS recalculation runs automatically after:
  - SITS import (Import SITS / Setup Wizard)
  - Sortie logging
  - Work package create / delete / close / remove task
  - Extension / deferral apply
  - Manual RunRefreshSITS

Step 8 — Work packages (Phase 4)
--------------------------------
Create:
1. On the SITS view, tick one or more Selector checkboxes.
2. Click **Create Work Package**.
3. If associated `/1` tasks exist on the **same Serial**, confirm the claim message.
   Same task code on a different Serial (e.g. the other engine) is not claimed.
4. Enter ORN in format `####TAILDDMMYY` (e.g. `7034ZZ383081125`):
   - First 4 digits: any number
   - Middle: must match the aircraft tail number
   - Last 6: valid calendar date DDMMYY
5. Selected + associated tasks are assigned the ORN; selectors are cleared.

Manage:
1. Click **Manage Work Packages** on the SITS sheet (or Dashboard **Work Packages**).
2. Read-only open count + dropdown of open ORNs.
3. Selecting an ORN lists Task Code + Task Description.
4. **Delete** — undoes the WP only (clears In Work ORN; no SITS reforecast).
5. **Remove task** — releases one selected task from the open WP (no reforecast).
   If the WP becomes empty it is deleted.
6. **Close** — completes all tasks in one go:
   - Reforecasts by Actual Interval (HH/E1/E2 hours, DD date, counters by number)
   - Auto-includes associated tasks already in the WP
   - Deletes PT2/PT3 deferred fault rows from SITS
   - Resets Extension to 0 / 0:00
   - Writes a Task Description cell comment: ORN, close date, user
   - Zero-interval tasks: skip auto-reforecast, flag Needs Manual Update (light blue + Dashboard count)
   - Rebuilds / re-sorts the SITS view so newly reforecasted tasks move to their
     correct due-soon position (e.g. 25:00 no longer sits above sooner jobs)
7. Form stays open and refreshes after Delete/Remove/Close.

Dashboard (Phase 4/5 / v2.3.1):
  Open work packages count
  Tasks needing manual update count
  Tasks in warning period count (amber — same rules as SITS view)
  Tasks overdue count (pink — same rules as SITS view)
  Status values in C6:C11 are centred
  Pre-setup / master: only **Setup Wizard** under User Actions
  After setup: Log Sortie, Work Packages, View Audit Log, Extend, Defer (two columns)
  Import SITS and Refresh Usage moved to Developer Tools

Developer Tools (hidden by default):
  Alt+F8 → **EnableDevTools** to show the whole section; **HideDevTools** (or the
  Hide Dev Tools button) to hide it again.
  Import SITS, Refresh Usage, Repair Formats, Repair Visibility, Validate Structure,
  Rebuild Dashboard, Ensure Missing Buttons, Phase 0 Setup, Rebuild SITS Layout,
  Format Usage Matrix, Hide Dev Tools

Step 9 — Extensions and deferrals (Phase 5)
-------------------------------------------
Extend (scheduled tasks — not PT2/PT3):
1. On the SITS view, tick one or more Selector checkboxes.
2. Click **Extend** (SITS sheet or Dashboard).
3. Validation before the form opens:
   - All selected tasks must share the same Interval Type Code
   - All selected tasks must share the same Actual Interval
   - PT2/PT3 cannot be extended (use Defer)
   - Tasks in an open work package (IN WORK) are blocked
4. Enter:
   - Extension amount (format matches interval: h:mm for HH/E1/E2, days for DD, number otherwise)
   - SNOW authorisation (required)
   - Authoriser name (required)
5. Warnings if total Extension (existing + this increment) exceeds 10% or 20% of Actual Interval.
6. On apply:
   - Adds the amount to any existing Extension
   - Writes Extension SNOW
   - Bumps baseline / life remaining by the increment
   - Appends Task Description cell comment (amount, SNOW, authoriser, date, user)
   - Clears selectors, recalculates SITS, refreshes the view
   - Audits EXTENSION_APPLIED

Defer (PT2/PT3 only — one at a time):
1. Tick exactly one PT2-D / PT2-H / PT3-D / PT3-H on the SITS view.
2. Click **Defer**.
3. Validation:
   - Exactly one selected task
   - Must be a deferred fault code
   - Must not be IN WORK
4. Enter:
   - PT2/3-H: airframe hours due as h:mm — life remaining = due − current HH
   - PT2/3-D: date due as dd/mm/yyyy — due date / life remaining set from that date
   - SNOW + authoriser (required)
5. On apply:
   - Sets life remaining / due from the entered due point
   - Records Extension delta + Extension SNOW
   - Appends Task Description cell comment
   - Clears selector, recalculates, refreshes
   - Audits DEFERRAL_APPLIED

Step 10 — Save
--------------
Save the workbook.

What Phase 5 includes
---------------------
- modExtensions — Extend / Defer apply lifecycle
- frmExtend — multi-task extension with amount + SNOW + authoriser
- frmDefer — single PT2/PT3 deferral with HH due or date due
- SITS buttons: Extend, Defer
- Dashboard buttons: Extend, Defer
- Same Interval Type + Actual Interval validation for Extend
- Authority warnings at >10% / >20% total Extension of Actual Interval
- Extension amount added to existing Extension; baseline LR bumped
- Deferral life remaining from absolute HH due or date due
- Task Description cell comments for SNOW + authoriser
- Remove single task from open work package (frmWorkPackage)
- Audit: EXTENSION_APPLIED / DEFERRAL_APPLIED / WP_TASK_REMOVED

Manual Add / Remove Task (SITS view)
------------------------------------
- modSITSEdit — Add Task / Remove Task entry points
- SITS buttons (H3/J3): Add Task, Remove Task (ensured on Refresh View; layout preserved)
- Add Task: sequential InputBox prompts for import-required fields
  (Interval Type Code, Life Remaining, Task Code, Task Sequence,
  Task Description, Serial, Actual Interval, Extension, End Item Serial).
  End Item Serial defaults to the configured tail and must match it.
  System-only columns (Selector, ORNs, completion fields, baselines,
  SITS_RowID, Needs Manual Update, current usage, optional import extras)
  are not prompted. Summary confirmation precedes insert; new rows appear
  on the view only if they pass the current forecast filter.
- Remove Task: uses existing Selector checkboxes. If none selected, prompts
  to select first. Confirmation lists all selected tasks and warns the
  delete is irreversible. IN WORK rows have no checkbox and cannot be
  selected. Orphan tblWPTasks links for deleted RowIDs are cleaned up.
- Audit: SITS_TASK_ADDED / SITS_TASKS_REMOVED

What is NOT included yet
------------------------
- Protected-sheet manual life-remaining edit form for zero-interval tasks
- Sortie correction / reversal UI (CorrectedFrom column reserved)
- Forecast settings UserForm (values editable on SITS sheet B3/B4/B5 for now)

SITS life remaining model (Phase 3)
-----------------------------------
GOLDesp life remaining at import is the **baseline** (trusted — may include
life accrued on other airframes). After import, each sortie increments usage;
life remaining is updated by **usage delta since the import snapshot**:

  Life Remaining = Baseline Life Remaining − (Current Usage − Snapshot Usage)

Current usage is looked up from tblUsageCurrent by **interval type code only**.
DD calendar tasks prefer Projected Due Date (absolute) when present — Life
Remaining is due date − today (can be negative when overdue). If only a day
count was imported, that count is reduced by elapsed days since the SITS
snapshot date.

On WP close, baseline life remaining is adjusted so RecalculateSITS yields the
full Actual Interval immediately after close (until further usage accrues).

On Extend, baseline life remaining is increased by the extension increment so
RecalculateSITS reflects the added life. On Defer (PT2/PT3-H), life remaining
is set from airframe HH due − current HH. On Defer (PT2/PT3-D), enter a date
due; Life Remaining becomes days remaining (date due − today) and Projected
Due Date is set to that date. For DD tasks, Projected Due Date is also updated
on extension and WP close reforecast.

Import rules (strict — abort if any required column missing)
------------------------------------------------------------
SITS Long Forecast:
  Interval Type Code, Life Remaining, Task Code, Task Sequence,
  Task Description, Serial, Actual Interval, Extension, End Item Serial
  Number-based interval types (excluding DD) are skipped on import unless both
  Actual Interval and Life Remaining are zero.

Usage exports (airframe + engine):
  Interval Type, Task Code, Life Used, Top Serial

Engine usage must include SM and SN interval codes after merge
(typically WC/STARTS/ECU on P4N155/P4N175).

Upgrading an existing workbook for the Log Sortie form redesign (v2.3.0)
------------------------------------------------------------------------
1. Replace modules: CONFIG, modUtility, modUsageLog, modSITSEdit,
   Setup_CreateDashboard (and paste updated BUILD notes as needed).
2. Insert > Class Module, Name = clsSortieUsageRow; paste
   vba/forms/clsSortieUsageRow.txt.
3. Recreate or update frmLogSortie:
   - Remove old list/pending controls (lstCounters, lstPending, etc.)
   - Add static shell per LOG_SORTIE_FORM_LAYOUT.txt
     (txtHumsOpNumber replaces txtSortieRef; add fraAirframe/fraEngines/fraWare)
   - Paste vba/forms/frmLogSortie.txt
4. Compile VBAProject.
5. Ensure usage imports include any catalog codes you need to log
   (WARE IL/IR/HE/FT were missing from older sample exports).
6. Usage sheet matrix (optional one-time):
   - Insert setup module modSetupUsageMatrix from
     vba/setup/Setup_FormatUsageMatrix.txt
   - Replace modUsageLog with the matrix-aware RefreshUsageView
   - Run **Setup_FormatUsageMatrix** (Dashboard developer button
     "Format Usage Matrix", or macro). Afterwards Refresh Usage / Log Sortie
     keep the matrix current.

Upgrading an existing Phase 0–4 workbook to Phase 5
-----------------------------------------------------
1. Import new modules: modExtensions, modSITSEdit, and updated modules (modSITS,
   modWorkPackage, CONFIG, Setup_CreateDashboard, Setup_RunAll).
2. Create frmExtend and frmDefer from the layout + code files.
3. Update frmWorkPackage controls to include cmdRemoveTask (or recreate the form).
4. Prefer **Ensure Missing Buttons** (and Refresh SITS) so new Extend / Defer /
   Add Task / Remove Task buttons appear without wiping your Master layout.
   Use **Rebuild Dashboard** only if you want a full layout reset.

Troubleshooting
---------------
"Expected variable or procedure, not module"
  -> Setup module Name must be modSetupRunAll etc., not Setup_RunAll.

"Variable not defined: frmLogSortie" / "frmWorkPackage" / "frmExtend" / "frmDefer"
  -> Create the UserForm in the IDE. The form object must exist even though
     the code is stored as a .txt file in the repo.

"Missing named range: Open_WP_Count"
  -> Run Setup_CreateNamedRanges or Rebuild Dashboard after pasting Phase 4+ VBA.

VBA name shadowing ("expects an array" on a function call)
  -> Do not name parameters the same as functions (VBA is case-insensitive).
     Example: use skipOverwritePrompt, not confirmOverwrite with ConfirmOverwrite.

All worksheets stuck hidden
  -> Click Repair Visibility on the Dashboard, or run RepairSetupSheetVisibility.

Phase 0 Setup fails when data sheets are hidden (error during "Building sheets")
  -> Fixed in modUtility / Setup_RunAll: setup now unhides all worksheets before
     deleting/rebuilding. Re-import those modules if an older build still fails.
  -> Workaround on an old build: Repair Visibility (or unhide all sheets), then
     run Phase 0 Setup again.

Serial numbers show as 1.11E+12 or lose leading zeros
  -> Re-import SITS/usage after updating modUtility, or click Repair Formats.
  -> Serial columns are stored as text (@) using the source cell display text.

Time values calculate incorrectly / show as 86.61 or plain text
  -> 86.61 is the internal day-fraction for 2078:43. Hour counters are stored
     as day fractions with [h]:mm (same principle as SITS), not as text.
  -> Click Repair Formats on the Dashboard, or re-run setup wizard import.
  -> HH/E1/E2 Format column will show Hours; Baseline and Usage show [h]:mm.

SITS life remaining does not change after sorties
  -> Re-import SITS or run Setup Wizard to refresh tblUsageSnapshot.
  -> Run RunRefreshSITS on the SITS sheet.

Dashboard buttons missing after updating VBA
  -> Click Ensure Missing Buttons (adds only what is missing).
  -> Use Rebuild Dashboard only if you want to reset column widths / placement.

Column widths / buttons reset after Refresh
  -> Refresh is data-only. If widths still reset, re-import updated modSITS /
     modUsageLog / Setup_CreateDashboard and avoid Rebuild Dashboard unless intended.

Audit Log columns resize after each log entry
  -> Re-import updated modAuditTrail (preserves widths around each append).
  -> Set preferred widths on the Master once; they carry to aircraft copies.

ORN rejected on create
  -> Must be ####TAILDDMMYY with this aircraft's tail and a real DDMMYY date.

Extend rejected — mixed intervals
  -> All ticked tasks must share Interval Type Code and Actual Interval.

Extend rejected — PT2/PT3
  -> Use Defer for deferred faults.

Defer rejected — more than one task
  -> Defer exactly one PT2/PT3 at a time.
